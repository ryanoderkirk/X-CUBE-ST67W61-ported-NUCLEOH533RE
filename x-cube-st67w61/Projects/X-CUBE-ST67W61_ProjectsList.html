X-CUBE-ST67W61 Expansion Package Projects list

Copyright 2024 STMicroelectronics

The X-CUBE-ST67W61 package comes with a set of applications and demonstrations running on STMicroelectronics boards, organized by board and provided with preconfigured projects for the main supported toolchains.

The Projects are located under X-CUBE-ST67W61_VX.Y.Z\Projects\, and all of them have the same structure:

  • /[Project Name]/Appli/App folder that contains the Wi-Fi/BLE/Net/MQTT/HTTP/FOTA application code.
  • /[Project Name]/Appli/Target folder that contains the Logging, Shell, Low-Power and BSP interfaces.
  • /[Project Name]/Core/Src folder that contains all STM32CubeMX generated sources code.
  • /[Project Name]/Core/Inc folder that contains all STM32CubeMX generated header files.
  • /[Project Name]/ST67W6X/App folder that contains the entry point to start the application associated to the ST67W6X_Network_Driver Middleware.
  • /[Project Name]/ST67W6X/Target folder that contains the configuration and port files to manage the ST67W6X_Network_Driver Middleware.
  • /[Project Name]/EWARM folder contains the preconfigured project for the IAR Embedded workbench for Arm.
  • /[Project Name]/MDK-ARM folder contains the preconfigured project for the RealView Microcontroller Development Kit.
  • /[Project Name]/STM32CubeIDE folder contains the preconfigured project for the STM32CubeIDE toolchain.
  • /[Project Name]/readme.md describing the example behavior and the environment required to run the example.

The projects are classified depending on the level they apply to, and are named as follows:

  • Applications intends to demonstrate the product performance and how to use the different Middleware stacks available. The Applications are organized per product feature that need high level firmware bricks. Integration of applications that use several Middleware stacks are provided as well.
  • Demonstrations aims to integrate and run the maximum of peripherals and Middleware stacks to showcase the product features and performance.

To run the example, you have to do the following:

These project require to use the ST67W611M Coprocessor binary st67w611m_mission_v2.0.89.bin.
Please follow the NCP Binaries README.md instructions using the NCP_update_mission_profile.bat script.

  • Open the example using your preferred toolchain.
  • Rebuild all files and load the image into target memory.
  • Run the example by following the readme instructions.
  • Note: refer to section "Development Toolchains and Compilers" and "Supported Devices and EVAL, Nucleo and Discovery boards" of the Firmware package release notes to know about the SW/HW environment used for the Firmware development and validation. The correct operation of the provided examples is not guaranteed on some environments, for example when using different compiler or board versions.

The provided Projects can be tailored to run on any compatible hardware.
User must to adapt the SPI configuration and align GPIO (SPI_CS, SPI_RDY, BOOT, CHIP_EN, USER_BUTTON) signals with the X-NUCLEO-67W61M1 Arduino connectors.

Projects Description

The table below contains the list of Projects provided within X-CUBE-ST67W61 Expansion Package.

In this table, the label CubeMX means the projects have been created using STM32CubeMX initialization code generator.
Those projects can be opened with this tool to modify the IP, Clock domain and Middlewares configuration including the ST67W6X_Network_Driver delivered in X-CUBE-ST67W61 Expansion Package.

The ioc files are generated for the STM32CubeMX 6.15.0 version.
